Understanding Dimensional Modelling

Dimensional modelling is a technique used to structure data warehouses for efficient querying and analysis. It separates business process data into facts and dimensions【885987071359135†L153-L158】.

Star Schema

The star schema is one of the most common architectures in dimensional modelling. It arranges data into a central fact table connected to dimension tables, forming a star‑like shape【885987071359135†L138-L147】. Fact tables store measurable, quantitative data, while dimension tables contain descriptive attributes that provide context to facts【885987071359135†L153-L158】.

Data Vault

While star schemas are great for simplicity and query performance, data vault modelling addresses scalability and auditability. A data vault is designed for long‑term historical storage of data from multiple operational systems【968585224980313†L155-L162】. It separates structural information (hubs and links) from descriptive attributes (satellites), making the model resilient to change【968585224980313†L174-L178】.

Choosing the Right Model

Choosing between star and data vault depends on your requirements. Use a star schema when you need fast query performance and a simple model for end‑users. Choose a data vault when you require flexibility, scalability and traceability across many source systems.
